# Contributing

Interface Behavior Lab is a speculative interaction research project. Contributions are welcome when they make a behavior clearer, more testable, more accessible, or more honest about its limitations.

## Good contributions

- A reduced-motion or alternate-input path
- A testable prototype of one behavior
- A critique grounded in usability, motor control, cognition, or accessibility
- A clearer state model or component contract
- A reproducible experiment comparing an adaptive behavior with a conventional control

## Avoid

- Decorative variants with no behavioral purpose
- Dark patterns disguised as “ethical friction”
- Motion that is required to understand state
- Claims that speculative input is universally available
- Apple branding, logos, or language implying official affiliation

## Pull request format

Include:

1. The problem being addressed
2. The behavioral hypothesis
3. The accessible fallback
4. What evidence would prove the idea wrong
5. Screenshots or a short recording when interaction changes

Keep the default control available as the comparison condition. The lab exists to test ideas, not to make every button sentient.
