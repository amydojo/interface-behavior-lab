# Interaction Lifecycle

Adaptive Controls organizes interaction around six moments rather than six visual styles.

```text
Idle → Approach → Intent → Commit → Resolve → Recover
```

## 1. Idle

The action is available but does not compete for attention. The control communicates its basic purpose and maintains a stable target.

## 2. Approach

The system senses proximity, focus, gaze, dwell, or another sign that the action may become relevant. Magnetic behavior may reduce motor effort, but it cannot change the user’s destination or capture input unexpectedly.

## 3. Intent

The system has enough context to make the consequence more specific. An Intent control may expand “Done” into “Save to Journal,” expose a destination, or show the affected object.

## 4. Commit

Input crosses a meaningful threshold. Pressure and Ethical behaviors belong here. A deeper stage must represent a deeper level of the same understood action, not a surprise category change.

## 5. Resolve

The system communicates that it is listening, processing, complete, or unable to continue. Breathing behavior can express this rhythmically, while a textual state remains available.

## 6. Recover

The completed action becomes its own recovery path when reversal is possible. The target remains stable, the remaining time is announced, and the result can still be located after the window expires.

## Cross-family choreography

A single high-consequence action may move through several families:

| Moment | Family | Role |
| --- | --- | --- |
| Approach | Magnetic | Assist target acquisition |
| Clarify | Intent | Name the exact consequence |
| Weigh | Ethical | Explain human impact |
| Commit | Pressure | Cross a deliberate threshold |
| Resolve | Breathing | Communicate system state |
| Recover | Reversible | Preserve agency after action |

This choreography is not a requirement to use all six. Most actions should use zero or one adaptive family. Complexity must be earned by the problem.
